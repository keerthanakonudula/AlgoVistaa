Category 01

Sorting Algorithms

Rearrange a collection of items into order. The most fundamental family of algorithms in computer science.

→
Sort

Bubble Sort

Repeatedly swaps adjacent elements if they're out of order. Simple but slow — excellent for learning swap mechanics.

Ω(n) Θ(n²) O(n²)
→
Sort

Selection Sort

Finds the minimum element in the unsorted portion and places it at the front. Always makes exactly n–1 swaps.

Ω(n²) O(n²)
→
Sort

Insertion Sort

Builds the sorted list one item at a time by inserting each element into its correct position. Excellent for nearly-sorted data.

Ω(n) O(n²)
→
Sort

Quick Sort

Picks a pivot, partitions elements around it, then recursively sorts each side. One of the fastest algorithms in practice.

Ω(n log n) O(n²)
→
Sort

Merge Sort

Divides the array in half, sorts each half recursively, then merges them. Stable and guarantees O(n log n) in all cases.

Ω(n log n) O(n log n)

Searching Algorithms

Locate a specific element within a data structure, each with different speed and precondition trade-offs.

Search

Linear Search

Checks every element one by one. Works on unsorted data. Simple but slow for large collections.

Binary Search

Halves the search space each step. Requires sorted input. Used in databases, spell checkers, and DNS lookup.

Navigate nodes and edges to find paths, spanning trees, and connected components.
